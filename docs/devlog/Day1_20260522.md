**Day 1**
- Tasks: README 설계 완성 / OpenDART API 유효성 검증 테스트 / OpenDART 문서 다운로드 테스트 / K-IFRS 기준서 PDF 파일 다운로드

- Details: 직전에 다룬 FinalProject의 경우 LangGraph가 적용되었지만 단순한 Sequential 구조를 보였다. 이번 프로젝트의 주된 목표는 create_react_agent에 기반하여 Supervisor 패턴을 적용한 MultiAgent를 구현해보는 것이다. Backend로는 비동기 처리가 용이하고 Swagger UI를 지원하는 FastAPI를 선택하였고, Frontend로는 js를 대체할 수 있는 Streamlit을 채택하기로 하였다. OpenDART에서 요청가능한 사업보고서가 XML 형식을 취하므로 이를 처리하기 위해 BeautifulSoup을 활용할 계획이고, K-IFRS 기준서를 임베딩하여 RAG 질의를 수행하기 위해 ChromaDB를 이용한다. MCP 서버는 DART와 K-IFRS 두 개로 분리하여 구축할 예정이다.
