Day 5
- Tasks: graph.py, main.py 수정, 완성 / dart_server.py 의 docstring 설계에 맞추어 코드 작성.

- Details: 
    graph.py의 agent와 supervisor을 build_graph 함수로 통합하여 MCP tool(dart_tools, kifrs_tools)을 받을 수 있도록 재설계하였다. 기존 agent의 경우에 tools = [tool1, tool2] 와 같은 형식으로 불러오도록 하였는데 MCP 환경에서는 이러한 방식이 동작하지 않는다는 것을 알았다. 그래서 _pick 함수를 도입하여 BaseTool list가 agent에 들어갈 수 있게끔 만들었다.
    
    graph.py에서 model을 다시 불러오지 않고 llm_utils의 get_llm을 활용하게끔 수정.
    main.py의 StdioServerParameters의 경로 command="python"을 sys.executable로 변경. MCP 패키지가 설치되지 않은 시스템 파이썬이 호출되는 것을 방지하기 위해 절대경로로 변경.

    uvicorn app.main:app 으로 FastAPI 실행했을 때 Connection closed 에러가 지속적으로 발생하여 lazy 추가, loop asyncio 시도 등등 해보았으나 안고쳐짐. 원인은 kifrs_server.py 맨 아래에
if __name__ == "__main__":
    mcp.run(transport="stdio")
를 누락했다는 것을 발견. MCP 서버 정상 호출 완료...

- Learn: 
    dart_tools == [
    <BaseTool name="search_company">,
    ...,
    ]
    load_mcp_tools 를 호출하면(await load_mcp_tools(dart_session)) 위와 같은 형식의 tool이 반환된다. 이를 dart_by_name으로 딕셔너리 형태로 변환하고 _pick 함수에 넣으면 <BaseTool name="search_company"> 가 agent의 tool로 들어가게 된다. 비로소 제대로 툴을 활용할 수 있게 됨.

    lazy 초기화: 툴이 실제로 호출되는 순간에만 빌드하고 두번째부터는 캐시된 객체를 그대로 재사용하는 방법. 초기화 비용이 무겁거나 툴이 안쓰일 수도 있는 경우에 사용한다. 

    @asynccontextmanager: __aenter__과 __aexit__ 없이도 비동기 컨텍스트 매니저를 만들 수 있게 한다. yield가 하나 들어간 함수에서 사용가능.
