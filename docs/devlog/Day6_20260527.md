**Day6**
- Tasks: dart_server.py의 tool을 스크립트 설계에 맞춰 구현. build_industry_cache.py 구축.

- Details:
    fetch_peers 툴의 경우 기존에 OpenDART의 산업코드 목록을 전부 파싱하여 동종업종간 비교를 수행하려고 설계했으나 KRX의 산업분류를 이용하기로 변경함. 이유는 편리성과 적합성. 산업분류 데이터를 불러오는 시간이 10분 -> 2초로 단축되고 또한 기존의 KSIC는 통계용 산업분류(예: 반도체 제조업)로서 세분이 매우 좁은 반면 KRX 업종분류(예: 반도체)는 투자자 관점의 분류에 가까워 이용자에게 더 적합한 정보를 제공할 수 있다고 판단함. parse_business_report_xml 툴을 구현하기 위해 HTML/XML 파싱 라이브러리 BeautifulSoup을 학습함.

- Learn:
    BeautifulSoup: HTML, XML의 복잡한 태그(<div>, <p>, <a> 등)들을 파싱된 트리 구조 객체로 다룰 수 있게 해주어 데이터를 쉽게 추출할 수 있게 하는 라이브러리. 태그를 찾기 위해 find(), find_all() 또는 CSS selector를 활용하는 select() 메서드를 활용할 수 있다.
